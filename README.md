# projeto-dio.me1

Contexto do Projeto
No cenário atual de suporte técnico e administração de dados, a lentidão em consultas (queries) é um dos principais "gargalos" que afetam a experiência do usuário final e o custo de infraestrutura. Este repositório é um Caderno Temático Digital dedicado ao estudo de técnicas avançadas de monitoramento, indexação e tunning de bancos de dados relacionais (com foco em Oracle e PostgreSQL).

Objetivos de Estudo
Identificação de Gargalos: Aprender a interpretar planos de execução (Execution Plans) para identificar Full Table Scans desnecessários.

Estratégias de Indexação: Compreender quando utilizar índices B-Tree, Bitmap ou índices compostos para reduzir o tempo de resposta.

Engenharia de Prompts para DBA: Desenvolver um conjunto de comandos (prompts) que ajudem a IA a analisar scripts SQL complexos e sugerir refatorações baseadas em boas práticas.

Automação de Diagnóstico: Criar um guia de scripts (SQL/Shell) que automatize a coleta de estatísticas de tabelas e fragmentação de índices.
